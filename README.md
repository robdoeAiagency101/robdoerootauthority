## 🏛️ Architecture

> Full technical deep-dive: **[ARCHITECTURE.md](ARCHITECTURE.md)**

The system is built on three cryptographic layers driven by 14-engine Byzantine consensus:

1. **Signal Layer** — Raw satellite frames ingested from BOM, Himawari-8, GOES-16, and Meteosat
2. **Decomposition Layer** — SHA256 tile hashing and cryptographic fingerprinting of every grid tile
3. **Witness Layer** — XYO bound-witness mesh + RFC3161 GPS-backed timestamping + immutable ledger anchoring

**Consensus gate:** 14 engines · Supermajority 10/14 required · K ≥ 0.99 threshold — execution gates open only when K ≥ 0.99.

See [ARCHITECTURE.md](ARCHITECTURE.md) for the full engine ring design, persistence layer, and API gateway specification.

## 🔐 Security & Trust

(Your security section preserved.)

- **SHA256 Tile Hashing:** Every satellite grid tile is independently hashed; any pixel-level tampering invalidates the hash.
- **HMAC-SHA256 Witness Signatures:** Each satellite observer node signs its observation, cryptographically binding the hash to the observer identity.
- **RFC3161 Timestamping:** GPS-backed timestamp authority binds the signed hash bundle to an independently verifiable moment; backdating is cryptographically impossible.
- **Append-Only Ledger:** The XYO witness ledger is write-once — no entry can be modified or deleted without breaking the chain of custody.
- **Multi-Satellite Consensus:** Independent observations from BOM, Himawari-8, GOES-16, and Meteosat must converge before a tile is accepted.
- **14-Engine Byzantine Fault Tolerance:** Decisions require K ≥ 0.99 coherence across 14 distributed engines; a single compromised engine cannot alter consensus.
- **Non-Root Containers & Network Policies:** All services run as non-root; Kubernetes network policies restrict inter-service traffic to declared paths only.

Full threat model and cryptographic specification: [SECURITY.md](SECURITY.md)

## 🎓 Academic Foundation

(Your research foundation preserved.)

The system design is grounded in established cryptographic and distributed-systems principles:

- **Byzantine Fault Tolerance (BFT):** PBFT-style consensus (Lamport, Shostak & Pease, 1982; Castro & Liskov, 1999) ensures correctness when up to ⌊(n−1)/3⌋ nodes behave arbitrarily.
- **Hash-chained append-only ledgers:** As pioneered in Certificate Transparency (Laurie et al., RFC 6962), hash-linked records provide tamper-evidence without requiring a central authority.
- **RFC3161 Trusted Timestamping:** The Internet X.509 PKI Time-Stamp Protocol (Adams et al., RFC 3161) provides externally verifiable, GPS-backed time anchoring.
- **XYO Cryptonetwork:** Proof-of-origin bound-witness protocol provides location-aware, decentralized attestation without a trusted third party.

See [ARCHITECTURE.md](ARCHITECTURE.md) for implementation details and [SECURITY.md](SECURITY.md) for the full cryptographic specification.
